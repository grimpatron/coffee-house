Data Transfer

Importance for Frontend Developer

  • Interaction with the server: Knowledge of the data transmission process helps in optimizing requests and responses between the client and the server.
  • Performance: Understanding network delays and network bandwidth allows you to improve the speed of web page loading.
  • Security: Knowledge of protocols and methods of data encryption ensures the protection of confidential information.
  • Debugging: The ability to work with network tools helps in diagnosing and troubleshooting network-related problems.

Internet

The Internet is a global system of interconnected computer networks for storing and transmitting data. It represents a ‘network of networks’, where millions of devices exchange information through standardized protocols.

Data communication

Data communication is the process of sending and receiving data between two or more devices over the Internet.

Data is transmitted in the form of packets through various physical and virtual communication channels using network protocols that define the rules for the exchange of information. Protocols ensure reliability and security of data transmission.

Network protocol

A network protocol is a set of rules that define how devices on a network interact. For sending and receiving information to be successful, all devices participating in the process must accept the terms of the protocol and follow them.

There are many data transmission protocols, each of which has its own features and scope. For example, HTTP and HTTPS are used for data transmission on the Internet, FTP allows you to transfer files, and SMTP, POP3, and IMAP are responsible for email operation.

Network protocols

  • HTTPS (HTTP Secure): An extension of HTTP with encryption for secure data transfer1.
  • FTP (File Transfer Protocol): Allows you to transfer files over the Internet1.
  • SMTP (Simple Mail Transfer Protocol): Used to send email1.
  • POP3 (Post Office Protocol version 3): Protocol for receiving email from a mail server1.
  • IMAP (Internet Message Access Protocol): Allows you to manage email on the server1.
  • DNS (Domain Name System): Converts domain names to IP addresses1.
  • TCP (Transmission Control Protocol): Provides reliable data transfer between devices1.
  • UDP (User Datagram Protocol): Used for fast data transfer without delivery confirmation1.
  • SSH (Secure Shell): A protocol for secure remote access to servers1.
  • SMTPS (SMTP Secure): A secure version of SMTP for sending mail over SSL or TLS1.
  • SNMP (Simple Network Management Protocol): Used to monitor and manage network devices1.
  • HTTP/2: Improved version of HTTP with support for multiple streams and header compression1.
  • ICMP (Internet Control Message Protocol): Used to exchange network error and status messages1.
  • RDP (Remote Desktop Protocol): Allows you to remotely control your computer or server1.

Network model

A network model is an abstract system that describes how devices in a network interact with each other.

One of the most well-known network models is the OSI (Open Systems Interconnection) model. It was developed to standardize interaction between different systems and consists of seven levels.

TCP/IP model

TCP/IP model: Has only 4 layers: application, transport, network and physical. More practical and widely used on the Internet. Based on TCP and IP protocols.

OSI model vs TCP/IP model

OSI model vs TCP/IP model

  • OSI has 7 layers whereas TCP/IP has 4 layers.
  • The OSI header is 5 bytes in size, while the TCP/IP header is 20 bytes in size.
  • OSI refers to Open Systems Interconnection, whereas TCP/IP refers to Transmission Control Protocol.
  • OSI follows a vertical approach, whereas TCP/IP follows a horizontal approach.
  • The OSI model, the transport layer, is connection-oriented only, while the TCP/IP model is both connection-oriented and connectionless.
  • The OSI model is developed by ISO (International Organization for Standardization), whereas the TCP model is developed by ARPANET (Advanced Research Projects Agency Network).
  • The OSI model helps standardize the router, switch, motherboard and other equipment, while TCP/IP helps establish connections between different types of computers.
